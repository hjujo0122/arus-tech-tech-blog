最初に作ったブログのデザインはNext.jsのデフォルトに近いもので、特にテーマはありませんでした。せっかくキャラクター（知佳ちゃん）がいるので、その雰囲気に合わせたデザインにしたいと思いました。

## 知佳ちゃんとは

このブログに登場するオリジナルキャラクターです。ヘッダーと、フッターから覗いているのが彼女です。

名前は「知佳（ちか）」。雷の髪飾りが特徴で、清楚で儚い雰囲気があります。ぼーっとしてる感じも好きです。

ブログのどこかに置きたくてイラストを用意しました。この子に合わせてデザインを作っています。

## 方針を決める

知佳ちゃんの印象をキーワードにしました。

> 清楚・儚さ・ぼーっとしてる・雷の髪飾り

これをもとに `specs/features/design-theme.md` を書きました。彩度を落としたラベンダーグレーを基調にして、コントラストは控えめ、shadowなし、フラットに保つ方針にしました。

## 背景色で3往復した

最初の背景色は `#f8f7fa` で、ほぼ白でした。全然ラベンダーじゃない。

`#eee9f8` に変えましたが、やはり薄い。「大胆に変えて」と指示したら `#ddd4f8` になりました。今度は自分でも変えすぎたと感じて「ごめんね」と言いながら中間の `#e6ddf8` に落ち着きました。

背景色1つに3往復かかりました。「なんとなくいい感じ」を言葉にするのが思ったより難しい。

## デザイン要素を足していく

色だけでは地味だったので、細かいところも詰めました。

- **フォント**: Klee One（手書き感のある丸みあるフォント）
- **カードのボーダー**: `border-dashed` + `rounded-2xl`で破線の丸角
- **左アクセントバー**: カード左端に細い紫のライン
- **ホバー時のシャドウ**: 紫がかった `box-shadow`
- **装飾記号**: 記事タイトルの前に `✦`

Klee OneはGoogle Fontsからの取得でつまずきました。`next/font/google` はビルド時にフォントをダウンロードしようとするのですが、ビルド環境がGoogleのサーバーに繋がらず254件のエラーが出て失敗。CDNリンクをHTMLに直接書く方式に切り替えて解決しました。

## 知佳ちゃんの置き場所問題

ヘッダーには小さく置いていましたが、どこかもう1箇所あるとかわいいと思って色々試しました。

**一覧ページの上に大きく表示** → 違和感があって却下

**フッターにテキストの横に並べる** → 「違和感凄くない？ｗ」という感想が出て却下

**フッターの境界線から上半身を覗かせる** → 採用

絶対配置で `top: -65px` にすることで、足元がちょうどボーダーラインに揃います。2回失敗してからの3回目でようやく決まりました。

## 記事ページのスタイル

記事の中も手を入れました。h2に左ボーダー、h3にアクセントカラー、コードブロックに深紫背景を入れています。

```css
.prose h2 {
  border-left: 3px solid var(--accent);
  padding-left: 0.75rem;
}

.prose h3 {
  color: var(--accent);
}

.prose pre {
  background: #271d38;
  color: #e8e0f8;
}
```

ラベンダー系の背景に対してコードブロックだけ暗くすることで、メリハリが出ます。

## まとめ

仕様書に方針を書いておいても、実際に見てみないとわからないことが多いです。特に色は難しい。何度でもやり直せるので気軽に試せるのが救いです。
