ブログを作ってからしばらく経ったので、「ちゃんとGoogleにインデックスされてるの？」「表示速度は大丈夫？」が気になり始めました。調べたらいろいろ問題があったので直しました。

## Googleにインデックスされているか確認した

まず `site:arus-tech-tech.web.app` でGoogle検索してみたところ、16記事あるのに1記事しか出てきませんでした。

原因は **Google Search Console**（Googleの管理ツール）に登録していなかったこと。Search Consoleとは、自分のサイトをGoogleに認識させてクロール状況を管理するためのツールです。

登録してsitemap.xml（サイトの地図ファイル）を送信すれば、Googleが残りの記事も拾ってくれるはずです。今はクロール待ちの状態です。

## PageSpeed Insightsでスコアを計測した

**PageSpeed Insights**とは、Googleが提供するサイト速度計測ツールです。URLを入れると「このサイトはどのくらい速いか」をスコアで教えてくれます。

計測してみたら、最初の文字や画像が表示されるまでの時間（FCP）が **7.2秒** という壊滅的な数字でした。スマホで7秒待たされたらブラウザバックしますよね。

## Googleフォントをself-hostに切り替えた

このブログは「Klee One」というGoogle提供のフォントを使っています。もともとの設定はGoogleのサーバーにリクエストを飛ばして読み込む方式でした。

**問題点：** Googleへのリクエストがページ表示の完了を待たせてしまい、しかも使っていないフォントのCSSが59 KiBも読み込まれていました。

Next.jsには `next/font` という機能があって、これを使うとGoogleフォントを自分のサーバーから配信できます（self-host）。外部リクエストがなくなるので速くなります。

```ts
// Before: <link> タグで外部から読み込み
// After: next/font で自前配信
import { Klee_One } from "next/font/google";

const kleeOne = Klee_One({
  weight: ["400", "600"],
  subsets: ["latin"],
  variable: "--font-klee-one",
});
```

これだけで **FCP が 7.2秒 → 2.1秒** になりました。見た目は変わらず、フォントはちゃんとKlee Oneのままです。

## 全画像をWebPに変換した

画像が重すぎる問題もありました。とくにひどかったのがフィギュア記事の写真で、4枚合計で **約7.4 MB** もありました。

**WebP**（ウェブピー）とは、Googleが開発した画像フォーマットです。JPGやPNGより同じ画質でファイルサイズがずっと小さくなります。

Node.js の `sharp` というライブラリで一括変換しました。

| ファイル | 変換前 | 変換後 |
|---------|--------|--------|
| mika-shelf.jpg | 966 KB | 127 KB |
| cr-all-class-epic.png | 2,387 KB | 29 KB |
| cabbage-soft-standee.jpg | 993 KB | 106 KB |

全画像合わせると **約13 MB → 約470 KB** になりました。7分の1以下です。

## 画像の置き場所も整理した

ついでに、`public/images/` フォルダに散らかっていた記事用画像を `public/posts/記事名/` の形に移動しました。「この記事の画像はここ」と場所が決まっていた方が管理しやすいです。

## キャッシュの設定をした

最後に、Firebase Hosting（ホスティングサービス）のキャッシュ設定をしました。

**キャッシュ**とは、一度読み込んだファイルをブラウザに保存しておく仕組みです。2回目以降のアクセスでは保存済みのファイルを使うので、ダウンロードが発生しません。

設定前はJS・CSS・画像すべてのキャッシュが1時間しかありませんでした。設定後はJSとCSSが1年、画像が30日になりました。リピーターには体感がかなり改善するはずです。

## 最終的なスコア

改善後の計測結果です。

| 指標 | 改善前 | 改善後 |
|------|--------|--------|
| FCP（最初の表示） | 7.2秒 | **2.1秒** |
| Speed Index | 7.2秒 | **3.2秒** |

FCP と Speed Index は大幅に改善しました。体感速度はかなりマシになったはずです。

LCP（最大要素の表示）は日本語フォントの構造上の問題でまだ遅いままですが、体感速度に直結する FCP が大きく改善したのでひとまずよしとします。
