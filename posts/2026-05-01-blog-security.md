「セキュリティ観点でレビューして」と言ったら問題点を出してくれたのでやっておきました。

## 何が問題だったのか

### セキュリティヘッダーが未設定

HTTPレスポンスに「このサイトはiframeに埋め込まれるな」「ファイルの種類を勝手に判定するな」といった指示を出せるのがセキュリティヘッダーです。

設定していないと、悪意のある第三者のサイトにこのブログをこっそり埋め込まれたりする可能性があります（クリックジャッキングという手法です）。

Firebase Hostingの `firebase.json` に数行追加するだけで設定できます。

```json
"headers": [
  {
    "source": "**",
    "headers": [
      { "key": "X-Frame-Options", "value": "DENY" },
      { "key": "X-Content-Type-Options", "value": "nosniff" },
      { "key": "Referrer-Policy", "value": "strict-origin-when-cross-origin" }
    ]
  }
]
```

### JSON-LDの `</script>` エスケープ漏れ

SEO対応でGoogleの構造化データ（JSON-LD）を記事ページに埋め込んでいます。記事タイトルや概要をJavaScriptのscriptタグの中に `JSON.stringify` で書き出しているのですが、タイトルに `</script>` という文字列が含まれているとタグが壊れる可能性がありました。

記事は自分で書くので実害は限りなく低いですが、直すのも簡単なので対応しました。

```ts
// 修正前
JSON.stringify(jsonLd)

// 修正後（< を < に置換）
JSON.stringify(jsonLd).replace(/</g, "\\u003c")
```
